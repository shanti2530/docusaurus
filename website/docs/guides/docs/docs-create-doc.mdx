### Doc URLs {#doc-urls}

By default, a document's URL is based on its file path relative to the `docs` folder, with a few exceptions. The URL segments are generated from folder and file names, using lowercase and hyphens. For example, `docs/GettingStarted/intro.md` becomes `/gettingstarted/intro`.

#### Filename Exceptions
If a file is named one of the following, the file name is omitted from the URL:

- **Named as `index` (case-insensitive):** `docs/Guides/index.md`
- **Named as `README` (case-insensitive):** `docs/Guides/README.mdx`
- **Same name as parent folder:** `docs/Guides/Guides.md`

In these cases, the URL is simply `/Guides`, without `/index`, `/README`, or duplicate `/Guides`.

:::note
This convention matches the [category index convention](./sidebar/autogenerated.mdx#category-index-convention). However, the `isCategoryIndex` configuration does _not_ affect the document URL.
:::

- `slug` is appended to the doc plugin's `routeBasePath` (default: `/docs`).
- See [Docs-only mode](docs-introduction.mdx#docs-only-mode) to remove the `/docs` prefix from URLs.

:::note
You can use:

- **Absolute slugs:** `slug: /my-slug`, `slug: /` (starts with `/`)
- **Relative slugs:** `slug: my-slug`, `slug: ./../my-slug` (no leading `/`)

Absolute slugs create URLs from the site root. Relative slugs are resolved based on the doc's location.
:::

#### Root URL Example
To make a document available at the root (e.g., `https://docusaurus.io/docs/`):

```md
---
id: my-home-doc
slug: /
---

Lorem ipsum
```

#### Best Practices & Troubleshooting
- Use lowercase and hyphens in slugs for clean URLs (e.g., `my-guide` not `My Guide`).
- Avoid duplicate slugs; each slug must be unique across all docs.
- If a doc is not appearing at the expected URL, check for conflicting slugs or folder structure issues.
- Changes to slugs require rebuilding the site to take effect.
